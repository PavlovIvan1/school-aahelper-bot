# Admin mini-app

Простая статическая страница (без сборки) — Telegram Mini App для рассылок.
Отдаётся тем же процессом бота, что и HTTP API (`admin_api.py`) — один порт,
один процесс, без Vercel и без CORS: фронтенд и API на одном origin.

## Как запустить

Всё делается на сервере одной командой из корня проекта:

```bash
./run.sh
```

Она сама:
1. поднимает `docker compose up -d --build` (бот + admin API/webapp + `cloudflared`),
2. дожидается HTTPS-адреса тоннеля в логах `cloudflared`,
3. вписывает его в `.env` как `ADMIN_WEBAPP_URL`,
4. пересоздаёт контейнер бота, если адрес изменился.

Перед первым запуском в `.env` должны быть заполнены `BOT_TOKEN` и `ADMIN_IDS`
(Telegram ID администраторов через запятую).

## Как попасть в панель

Админ пишет боту `/admin` → приходит кнопка «🛠 Открыть панель» (Telegram
WebApp-кнопка, ведёт на `ADMIN_WEBAPP_URL`) → панель открывается прямо внутри
Telegram.

## Как это работает

- Telegram передаёт панели `initData` (подписанные данные пользователя).
- Бэкенд проверяет подпись по HMAC(bot_token) и сверяет `user.id` со
  списком `ADMIN_IDS` — так что доступ есть только у админов, без паролей.
- Рассылка идёт асинхронно с ограничением по скорости и обработкой блокировок
  бота пользователями; прогресс опрашивается страницей раз в 1.5 сек.

## Нюанс бесплатного cloudflared tunnel

Без своего домена `cloudflared` каждый раз выдаёт новый случайный
`*.trycloudflare.com` адрес — он меняется при каждом перезапуске контейнера
`cloudflared`. Просто гоняйте `./run.sh` заново после любого рестарта — он
сам подхватит новый адрес и обновит `.env`. Если захотите постоянный адрес —
понадобится завести бесплатный домен в Cloudflare и сделать именной tunnel.
