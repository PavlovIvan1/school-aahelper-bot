# Admin mini-app

Простая статическая страница (без сборки) — Telegram Mini App для рассылок.
Ходит в HTTP API бота (`admin_api.py`), который крутится вместе с самим ботом.

## Деплой на Vercel

1. Открой `config.js` и укажи `API_BASE_URL` — публичный HTTPS-адрес,
   по которому будет доступен `admin_api.py` (порт `WEBAPP_API_PORT`,
   по умолчанию 8080). Нужен реверс-прокси с HTTPS (nginx/Caddy) перед
   ботом — просто открыть порт недостаточно, Telegram Mini App и браузер
   требуют HTTPS.
2. Задеплой папку `admin-webapp/` на Vercel как статический проект
   (New Project → Import → выбрать эту папку как Root Directory, framework —
   Other/Static, build command не нужен).
3. В `.env` бота пропиши:
   - `ADMIN_IDS` — Telegram ID администраторов через запятую
   - `ADMIN_WEBAPP_URL` — выданный Vercel URL (например `https://xxx.vercel.app`)
   - `CORS_ALLOWED_ORIGIN` — тот же URL (если не задать, бот возьмёт его из `ADMIN_WEBAPP_URL`)
4. Перезапусти бота. Админ пишет боту `/admin` → получает кнопку,
   открывающую панель внутри Telegram.

## Как это работает

- Telegram передаёт панели `initData` (подписанные данные пользователя).
- Бэкенд проверяет подпись по HMAC(bot_token) и сверяет `user.id` со
  списком `ADMIN_IDS` — так что доступ есть только у админов, без паролей.
- Рассылка идёт асинхронно с ограничением по скорости и обработкой блокировок
  бота пользователями; прогресс опрашивается страницей раз в 1.5 сек.
